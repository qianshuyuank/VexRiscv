根据之前的建议，我们将采用 **“Windows 主机（安装 FPGA 工具）+ WSL2 Ubuntu（安装 VexRiscv 开发环境）”** 的混合模式。以下是分步配置指南：

---

## 第一阶段：Windows 主机设置 (FPGA 软件)

1.  **安装安路 Tang Dynasty (TD) IDE**：
    * 从安路官网或开发板资料包下载 TD 软件。
    * [cite_start]安装过程中请务必允许安装 **USB 驱动**，这是识别 MLKPAI 串口和 JTAG 的关键 [cite: 14]。
2.  **启用 WSL2** [cite: 1, 2]：
    * 以管理员身份运行 PowerShell，输入：`wsl --install` [cite: 1]。
    * 重启电脑。
    * 安装完成后，在 Microsoft Store 下载并打开 **Ubuntu 22.04 LTS** [cite: 3]。

---

## 第二阶段：WSL2 Ubuntu 环境配置 (核心依赖)

进入 Ubuntu 终端后，依次执行以下命令：

### 1. 更新系统并安装基础工具
```bash
sudo apt-get update
sudo apt-get install git make autoconf g++ flex bison bc libncurses5-dev libssl-dev -y
```

### 2. 安装 Java JDK 8
[cite_start]VexRiscv 依赖 Scala 环境，而 Scala 目前在 JDK 8 下运行最稳定 [cite: 2, 6, 13]。
```bash
sudo apt-get install openjdk-8-jdk -y
# 验证版本
java -version 
```

### 3. 安装 SBT (Scala Build Tool)
[cite_start]SBT 用于管理 VexRiscv 项目的构建 [cite: 2, 16]。
```bash
echo "deb https://repo.scala-sbt.org/scalasbt/debian all main" | sudo tee /etc/apt/sources.list.d/sbt.list
curl -sL "https://keyserver.ubuntu.com/pks/lookup?op=get&search=0x2EE0EA64E40A89B84B2DF73499E82A75642AC823" | sudo apt-key add
sudo apt update
sudo apt install sbt -y
```

### 4. 安装 Verilator (仿真工具)
[cite_start]这是进行 CPU 功能仿真所必须的工具 [cite: 2, 16]。
```bash
sudo apt-get install verilator -y
# 验证版本，建议在 4.x 以上
verilator --version
```

---

## 第三阶段：RISC-V 工具链安装 (编译 C 程序)

[cite_start]为了在 Murax SoC 上运行你的 C 代码，需要安装交叉编译器 [cite: 6, 16]。
你可以选择手动编译，但推荐直接下载预编译版本（如 SiFive 版）：

1.  下载压缩包（在 Ubuntu 中操作）：
    ```bash
    wget https://static.dev.sifive.com/dev-tools/riscv64-unknown-elf-gcc-8.3.0-2020.04.1-x86_64-linux-ubuntu14.tar.gz
    ```
2.  解压并添加环境变量：
    ```bash
    tar -xvf riscv64-unknown-elf-gcc-*.tar.gz
    sudo mv riscv64-unknown-elf-gcc-* /opt/riscv
    echo 'export PATH=/opt/riscv/bin:$PATH' >> ~/.bashrc
    source ~/.bashrc
    ```

## 第四阶段：获取 VexRiscv 源码

```bash
git clone --recursive https://github.com/SpinalHDL/VexRiscv.git
cd VexRiscv
# 测试是否能正常启动构建
sbt test
```

## 第五阶段：工作流打通 (Windows 与 WSL 共享)

[cite_start]建议在 Windows 的磁盘中创建一个文件夹用于存放生成的代码，以便 Windows 下的 TD IDE 能直接读取 [cite: 19]。

1.  在 Windows 的 D 盘创建 `Anlogic_Project` 文件夹。
2.  在 Ubuntu 中通过 `/mnt/d/Anlogic_Project` 访问该目录。
3.  [cite_start]后续生成 Murax.v 时，将其输出路径指向此处 [cite: 7, 19]。
